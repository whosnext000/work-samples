Welcome and thank you for purchasing the ProFolio plugin!

This guide should help you understand all there is to know about using the plugin. Since this is a single page guide, you can use Ctrl+F to search the entire document at any time if you like.

ProFolio is created entirely from scratch (excluding Fancybox) so other than Wordpress, there are no outside frameworks that you need to learn to use this plugin. Feel free to jump right in.

Installation & Setup

Important
Before installing the plugin, you need to unzip the file you downloaded from CodeCanyon which contains the necessary files, documentation and more. You can even unzip it directly into your wp-content/plugins directory if you like since the file structure is already set.

Here's what to do:

Make sure you have an FTP client installed on your computer, FileZilla is highly recommended. You'll need to use the FTP credentials provided by your host to access the files on your site.

  1. Put all files from the ProFolio folder in your wp-content/plugins folder.
  2. Login to your Wordpress install (eg: yoursite.com/wp-admin).
  3. Once logged in, navigate to p Plugins > Installed Plugins. ProFolio will be listed but greyed out (meaning it's not active).
  4. Click Activate to start using the plugin. You may be asked to enter your FTP credentials, your host should have provided that to you.
  5. Look in the navigation panel for ProFolio.

How to Use

In this section you'll gain an understanding of how to use the ProFolio Wordpress Plugin so you don't run into problems later.

If you're unsure of something, please reference this guide to help you, that's what it's here for. You can also ask in the support forums if you can't find your answer here.

Add Projects

Adding a project to your portfolio is fairly simple and straight forward. Most sections will have a tooltip with examples to make it even easier to understand.

Your Add Projects page will look like this:

  1. Your Project Title can be anything you want with any length.
  2. The Project Type can be anything from Web Design to Buisiness Plans to Criminal Cases. ProFolio is not limited to those in the web industry, it can even be used to list products for sale. In this case, Project Type would be like your product categories.
  3. Add a logo to identify the project or an image for the brand of product you're selling. For best results, your image should scale easily to a max width of 250 pixels and no more than 78 pixels in height.
  4. The Project Screenshot should be the best image that shows off the product or project. Similar to the logo, this image should scale easily to a maximum of 250 pixels in width and no max height.
  5. The Project Description should be as detailed as possible. Auto-formatting has been removed to allow maximum editability by using simple HTML tags such as <p>, <br /> and others.

  6. NEW - Turn On/Off Show Excerpt which will appear under the projects thumbnail. It's limited to 100 characters for now but as a highly requested feature, we threw it in for good measure.
  7. The Project Technologies section is used to let your viewers know what technologies were used during the creation of your project. This can be HTML, PHP, CSS, ASP, etc. It is recommended that your separate each item with a pipe "|" for easiest readability and so that they're read correctly in the front-end should you choose to use this section as a filter.
  8. The Client Name can be any client that you've added to the Manage Clients section.
  9. Project URL is the website address to the viewable project, whether on your site or elsewhere.
  10. The Project Year is the date in which the project was finished.

Manage Clients

In this section you can create, edit and delete each client associated with your projects.

Manage Projects

In this section you can edit and delete projects. You'll get an overview of your projects with the following:

  1. A preview of the projects Logo.
  2. The projects Title.
  3. The Client selected for the project.
  4. The Categor(y)ies (Project Type(s)) associated with the project.
  5. Whether the Status of the project is online or offline. Cannot be set in Add Projects.

Edit The Project

Tip: Online/Offline status can only be set when editing a project.

Everything that can be edited in Add Projects can be edited here with the exception of:

  1. Project Logo and Screenshot have previews of the uploaded images.
  2. The Project Status can be set and turned on or off.

Tip: The toggles are for turning things on/off. Green is for On and Red is for Off.

Manage Project Types

In this section you can create, edit and delte the Project Types. These are very similar to categories, only with a more fitting naming convention.

Project Types must have a Title AND Description to be saved.

Settings

Here's where the magic is. In the Settings section, you get various options for customizing the portfolio to look and feel the way you want, such as:

General Settings

  1. Select Theme is where you can select any pre-built theme or entirely create your own based off the default.css file. There are 7 themes included and 1 skeleton.
  2. Results Per Page allows you to control the pagination, so if you set it to 3 the user will be able to navigate to the next page (which uses Ajax) for every 3 items. 6 is the recommended max setting to decrease the amount of downward scrolling.

Filter Settings

  1. Create a New Filter is for the title of your filter.
  2. Select the Database Column that you would like to use as a Filter for your portfolio. Create a New Filter will overwrite the name of this column in the front end.
  3. In the list you'll get the custom name of your filter with the database column name. If you don't want to show a filter, it can simply be deleted.

We decided to allow the selection of a database column to give more control over what to display.

FancyBox Settings

As of now there is only one setting which allows you to display the title of your project in 3 different places in the Fancybox.

  1. Outside: Displayed outside of the Fancybox container
  2. Inside: Displayed inside of the Fancybox containter
  3. Over: Displayed anywhere in the Fancybox container.

Shortcodes

As of now, there is only one shortcode you need to remember and it can be placed anywhere inside the MCE editor.

Use the following shortcode:

  • [portfolio]

Files & Structure

After you've unzipped the files downloaded from CodeCanyon, you'll see that the file structure is quite simple.

Important
Do not edit any of the core files included in the plugin files.

The CSS Directory

Instead of editing the core files which you shouldn't do, you can add your own themes to the plugins css/theme directory. Simply copy the default.css file, rename it and start styling to your hearts content.

After you've created a new theme, you can choose it in the settings.

The images folder in the root is for core files. Notice the images folder in the plugins css/theme directory, this is where to put images for your theme.

The rest are all core files and should be left alone. Repeating this is necessary, you will overwrite any changes when upgrading.

Understanding default.css

If you look through the stylesheet first, it'll be easier to create your own theme. Each element is defined with #profolio-port at the beginning which helps ensure maximum compatability with your site. This practice should be continued in your custom theme for the plugin.

Tip: Searching for Mobile Compatability in the stylesheet will take you to the styles required for making sure your theme is responsive and compatible with mobile devices.

All elements are commented with appropriate names so they're easy to find, such as

#profolio-port .boundingBox is part of the /* Layout */ section.

How to Update

Occasionally there will be updates to the plugin in which case you'll want to go back to CodeCanyon and download a fresh copy.

You may want to check out the changelog to see what's new or find out what bugs have been fixed.

To update your plugin:

  1. Backup your files!
  2. Unzip the file you downloaded from CodeCanyon
  3. In your FTP Client, login with the FTP credentials your host provided.
  4. Navigate to where you unzipped the downloaded files, select them all and upload them to the wp-content/plugins directory.

Assuming you have not edited any of the core files, it is safe to overwrite all files when your FTP clients asks. Your custom portfolio themes will not be overwritten.

FAQs

The FAQ section will be expanded online as common issues come up.

How can I filter by price for products?

In the section for Settings > Filters, you can specify the name of the filter (ie: Price). The database column you may want to use is project_year where you can simply enter the price for your product in Add Projects.

Is it possible to change the text that says "Filter by Category", etc to something else?

Yes, in Settings > Filters you can set the name for your filter in the Create a New Filter field.

This plugin breaks my other plugins. How can this be avoided?

Please upgrade to version 2.0 by going to the product page in CodeCanyon and downloading it.

Version 2.0 and beyond should be compatible with nearly all plugins. If you find one that it breaks, please report it in our support forum.

I’m not able to add clients: I receive an error “You do not have sufficient permissions to access this page”.

This was a known issue prior to version 2.0 and has now been fixed. Please upgrade to version 2.0 by going to the product page in CodeCanyon and downloading it.

Some of my other jQuery plugins are not working. Why?

Wordpress 3.5 - 3.5.1 uses jQuery 1.8.3 which has caused numerous conflicts, so in these versions, an alternate version of jQuery is loaded outside of wp_enqueue. This may be causing conflicts depending on the order jQuery is loaded (make sure it's always the first script before any other jQuery scripts).

In versions before and beyond 3.5.x, ProFolio will revert back to the wp_enqueue to properly include the scripts.

My design is broken. What can I do?

Starting with version 2.0, the styles are conditional with

#profolio-port; the plugins wrapper, so not to conflict with other styles. Please upgrade to version 2.0 by going to the product page in CodeCanyon and downloading it.
